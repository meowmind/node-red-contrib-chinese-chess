## 一、FEN 格式处理

### 1.1 FEN 加载 Bug - 棋子叠加问题

**问题：**
- `loadFEN()` 先调用 `this.reset()` 摆好初始棋子，再在上面叠加 FEN 中的棋子，导致棋子数量翻倍
- 表象：4个炮、4个马、棋子消失等异常

**修复方案：**
```javascript
// 错误做法：
this.reset(); // 先摆初始棋子 → 再叠加FEN

// 正确做法：
// 重置棋盘，但不摆放初始棋子（完全清空）
this.board = new Board();
// 清空棋盘所有位置
for (var y = 0; y < 10; y++) {
    for (var x = 0; x < 9; x++) {
        this.board.squares[x][y] = null;
    }
}
this.history = [];
this.currentMoveIndex = 0;
this.recording = true;
```

### 1.2 FEN 合法性校验

**校验项：**
1. ✅ 行数必须为 10
2. ✅ 每行宽度必须为 9
3. ✅ 棋子数量限制（将1、士2、相2、马2、车2、炮2、兵5）
4. ✅ 将帅必须在九宫范围内
5. ✅ 非法字符合法性检查

**兼容模式：**
- 支持国际象棋记法自动转换：`N/n` → `马`，`B/b` → `象`
- 兼容模式可自动识别并转换不规范输入

### 1.3 FEN 提取功能

从任意字符串中提取合法 FEN（支持夹杂文本的输入），通过检查每行宽度是否为9来验证。

## 二、Canvas 视觉优化经验

### 2.1 棋盘绘制

**木纹效果：** 使用线性渐变 + 贝塞尔曲线纹理线条模拟真实木纹
```javascript
var boardGradient = ctx.createLinearGradient(...);
boardGradient.addColorStop(0, '#e8c99e');
boardGradient.addColorStop(0.3, '#deb887');
boardGradient.addColorStop(0.5, '#d4a76a');
boardGradient.addColorStop(0.7, '#deb887');
boardGradient.addColorStop(1, '#e8c99e');
```

**格线颜色：** 使用棕色 `#8b4513` 替代黑色，更符合传统棋盘风格

**边框粗细问题：**
- ❌ 错误：使用 `roundRect()` + `stroke()`，底部边框会和格线叠加变粗
- ✅ 正确：四条边框单独绘制，底部用细线（2px），其他三边用粗线（4px）

### 2.2 传统棋盘标记

炮位和兵位的四角三角形标记：
```javascript
// 绘制标记的辅助函数
function drawMarker(x, y) {
    var cx = padding + x * cellSize + cellSize / 2;
    var cy = padding + y * cellSize + cellSize / 2;
    
    // 边缘位置只画一半，不超出棋盘
    if (x > 0) { /* 画左侧两个三角形 */ }
    if (x < 8) { /* 画右侧两个三角形 */ }
}
```

**标记位置：**
- 黑方卒位：(0,3)(2,3)(4,3)(6,3)(8,3)
- 黑方炮位：(1,2)(7,2)
- 红方兵位：(0,6)(2,6)(4,6)(6,6)(8,6)
- 红方炮位：(1,7)(7,7)

### 2.3 棋子视觉优化

**棋子文字：**
- 字体：楷体 `KaiTi, STKaiti, serif`，书法感更强
- 区分：红方用单人旁繁体（傌、俥），黑方用纯繁体（馬、車），黑方炮用石字旁砲

**真实质感：**
- 多层径向渐变（高光、亮部、暗部、阴影）
- 外阴影增强立体感
- 白色内圆圈装饰边框
- 文字阴影增强可读性

### 2.4 选中效果

选中棋子时**放大**而非色块高亮，效果更自然：
```javascript
var isSelected = this.selected && this.selected.x === x && this.selected.y === y;
var pieceRadius = isSelected ? cellSize * 0.45 : cellSize * 0.40;
```

## 三、动画实现经验

### 3.1 移动动画架构

**状态管理：**
```javascript
this.animating = {
    piece: piece,        // 正在移动的棋子
    fromX: fromX, fromY: fromY,  // 起始位置
    toX: toX, toY: toY,          // 目标位置
    progress: 0,         // 动画进度 0→1
    startTime: Date.now()
};
```

**动画循环：** 使用 `requestAnimationFrame` (60fps)
```javascript
Renderer.prototype.startAnimationLoop = function() {
    var self = this;
    function animate() {
        if (self.animating) {
            // 更新进度
            var now = Date.now();
            var elapsed = now - self.animating.startTime;
            var duration = 200;
            self.animating.progress = Math.min(elapsed / duration, 1);
            
            if (self.animating.progress >= 1) {
                self.animating = null;
            }
            self.render();
        }
        self.animationId = requestAnimationFrame(animate);
    }
    this.animationId = requestAnimationFrame(animate);
};
```

### 3.2 缓动函数

使用 **easeOutQuad** 实现自然减速效果：
```javascript
var t = anim.progress;
var easeProgress = t * (2 - t); // 减速效果

var currentX = fromX + (toX - fromX) * easeProgress;
var currentY = fromY + (toY - fromY) * easeProgress;
```

**常用缓动函数参考：**
| 名称 | 公式 | 效果 |
|------|------|------|
| Linear | `t` | 匀速 |
| easeOutQuad | `t*(2-t)` | 减速（推荐用于移动） |
| easeOutCubic | `(t-1)^3+1` | 更明显减速 |
| easeOutBack | `...` | 回弹效果 |

### 3.3 动画绘制顺序

1. ✅ 先绘制棋盘背景、格线
2. ✅ 绘制移动轨迹虚线（`ctx.setLineDash([5,5])`）
3. ✅ 绘制静止棋子（动画中的棋子跳过原位置）
4. ✅ 在插值位置单独绘制正在移动的棋子

### 3.4 动画触发时机

**正确顺序：**
1. 先执行 `game.move()` 更新棋盘状态
2. 然后调用 `triggerMoveAnimation()` 传入移动前的棋子引用
3. 动画期间不绘制目标位置棋子，只在插值位置绘制

**错误做法：** 先画动画再执行走棋 → 状态不同步

## 四、前端交互经验

### 4.1 FEN 检测逻辑

**错误：** 依赖空格判断 FEN
```javascript
if (typeof msg.payload === "string" && msg.payload.indexOf(" ") > 0)
```

**正确：** 使用 `/` 检测（纯 FEN 也包含行分隔符）
```javascript
if (typeof msg.payload === "string" && msg.payload.indexOf("/") >= 0)
```

### 4.2 前端输出

导入 FEN 成功后通过 `move` 事件输出完整状态，和正常走棋保持一致：
```javascript
this.triggerEvent('move', {
    fen: this.getFEN(),
    history: this.history,
    currentStep: this.history.length,
    totalSteps: this.history.length,
    turn: this.currentMoveIndex % 2 === 0 ? 'red' : 'black'
});
```

## 六、经验总结

| 类别 | 经验 |
|------|------|
| ⚠️ **Bug 教训** | 1. `reset()` 会摆放初始棋子，加载 FEN 前必须先清空<br>2. Canvas `stroke()` 会叠加，相邻线条要注意粗细<br>3. 字符串检测不能依赖可选字符（如 FEN 的空格） |
| ✅ **最佳实践** | 1. Canvas 动画用 `requestAnimationFrame` 不用 `setInterval`<br>2. 动画要加缓动函数，不要用匀速<br>3. 视觉优先用放大/缩小等自然效果，而非色块高亮 |
| 🎯 **产品思维** | 1. 支持兼容模式，不要轻易拒绝用户输入<br>2. 象棋是传统文化，视觉要匹配传统风格<br>3. 动画时长 200ms 是用户体感舒适区间 |
| 📐 **设计原则** | 1. 边缘元素不要画到容器外<br>2. 装饰性元素要有边界判断<br>3. 同类操作输出格式保持一致性 |
