### ✅ 问题 1：兵的过河判断边界错误
**发现时间：** 2026-04-21 下午
**问题描述：**
- 兵走到楚河汉界后还不能左右横走，需要再前进一格
- 红方 `fromY > 4` 判定错误，导致 y=5 也算"未过河"
- 黑方 `fromY < 5` 判定错误，导致 y=4 也算"未过河"

**修复方案：**
```javascript
// 修复前
if (fromY > 4) { /* 红方未过河 */ }
if (fromY < 5) { /* 黑方未过河 */ }

// 修复后
if (fromY > 5) { /* 红方未过河 (y=6,7,8,9) */ }
if (fromY < 4) { /* 黑方未过河 (y=0,1,2,3) */ }
```

**修复文件：** `resources/chinese-chess-core.js` (line 190-209)

### ✅ 问题 3：中文记谱坐标方向错误
**发现时间：** 2026-04-21 23:35
**问题描述：**
- 发送"炮二平五"，实际走的是炮八平五

### ✅ 问题 4：双重转发导致重复输出
**发现时间：** 2026-04-21 23:45
**问题描述：**
- 发送一次 inject 输出两次相同消息
- 同时设置了 `forwardInputMessages: true` 和 `node.send(msg)`

**修复方案：**
```javascript
// 修复前
forwardInputMessages: true,
node.on('input', function(msg, send, done) {
    node.send(msg);  // 重复转发
});

// 修复后
forwardInputMessages: false,
node.on('input', function(msg, send, done) {
    // 不自动转发，由前端控制输出
});
```

**修复文件：** `nodes/chinese-chess.js`

### ✅ 问题 5：非法走法不输出报警
**发现时间：** 2026-04-21 23:45
**问题描述：**
- 非法走法（如起点无棋子、不符合规则）棋子不动，但不输出任何消息
- 只静默失败，无法在 flow 中捕获错误

**修复方案：**
```javascript
// 起点无棋子
$scope.send({
    payload: {
        error: "非法走法",
        message: moveText + " (起点无对应棋子)",
        success: false
    }
});

// 不符合规则
$scope.send({
    payload: {
        error: "非法走法",
        message: moveText + " (不符合规则)",
        success: false
    }
});

// 无法解析
$scope.send({
    payload: {
        error: "无法解析走法",
        message: moveText,
        success: false
    }
});
```

**信息面板同步添加 emoji 提示：**
- ✅ 执行走法成功
- ⚠️ 非法走法
- ❌ 无法解析走法

**修复文件：** `nodes/chinese-chess.js`

### ✅ 问题 6：中文记谱绕过轮次检查

- 但中文记谱接口绕过了这个检查，第一步发"炮2平5"（黑方）也能走

var expectedIsRed = (game.currentMoveIndex % 2 === 0);

### ✅ 问题 7：回退后轮次判断错误
**发现时间：** 2026-04-22 00:18
**问题描述：**
- 红走 → 黑走 → 点"上一步"撤销黑方走法
- 此时应该轮到黑方继续走，但实际显示轮到红方
- 原因是所有地方都用 `history.length` 判断轮次，但回退后 `history.length` 不变，只有 `currentMoveIndex` 减少了

**修复方案：**
统一把所有轮次判断从 `history.length` 改为 `currentMoveIndex`

| 位置 | 用途 |
|------|------|
| `chineseMoveToCoordinate` | 中文记谱轮次检查 |
| `getCurrentTurn` | 鼠标点击选子权限 |
| 输出 msg.turn 字段 | 告诉 Node-RED 轮到谁走 |
| 信息面板文字 | 界面显示"轮到红方/黑方" |

**共修复 4 处代码**

## 二、接口规范

### 输入格式
```javascript
// 中文记谱走棋
{"action": "炮二平五"}   // 红方：中文数字 一~九
{"action": "马8进7"}    // 黑方：阿拉伯数字 1~9

// 其他动作
{"action": "reset"}           // 重置棋盘
{"action": "prev"}            // 上一步
{"action": "next"}            // 下一步
{"action": "goto", "step": 0} // 跳转到指定步数
{"action": "loadfen", "fen": "..."} // 加载 FEN 局面
```

### 输出格式
**走法成功：**
```javascript
{
    payload: {
        fen: "...",          // 当前局面 FEN
        pgn: "...",          // PGN 棋谱文本
        pgnArray: [...],     // 着法数组
        currentStep: 2,      // 当前步数
        totalSteps: 2,       // 总步数
        turn: "red",         // 轮到谁走 red/black
        history: [...]       // 完整走子历史
    }
}
```

**走法失败：**
```javascript
{
    payload: {
        error: "非法走法",   // 或 "无法解析走法"
        message: "...",      // 详细原因
        success: false
    }
}
```
