# node-red-contrib-chinese-chess 开发日志

## 2026年4月25日 - 同列多棋子记谱功能开发

### 开发目标
支持中国象棋同列多棋子的中文记谱法（如"前车平七"、"前炮二平五"等）

### 问题分析
1. **输入解析问题**：原代码只支持标准记谱格式（如"炮二平五"），不支持"前"、"后"开头的方位词格式
2. **输出生成问题**：走子后生成记谱时，棋子已经移动，无法正确判断"前"、"后"
3. **边界情况**：棋子不在同列时不应该加前缀

### 解决方案演进

#### 第一版（输入解析）
- 检测"前"、"后"方位词前缀
- 支持4字符格式（前车平七）
- 支持5字符格式（前炮二平五）

#### 第二版（输出修复 - 走子后判断）
- 遍历棋盘找同类型棋子
- 问题：棋子已移动，需要"把起点棋子加回去"
- 问题：判断逻辑复杂，容易出错

#### 第三版（最终方案 - 走子前判断）
- **核心思路**：在 `game.move()` 走子前，先判断起点列是否有多个同类型同色棋子
- 走子前遍历棋盘，按列分组
- 如有多棋子，判断是"前"还是"后"，保存到 `move.positionPrefix`
- 生成记谱时直接使用保存好的前缀
- **优点**：逻辑简单，数据准确，无边界问题

### 代码改动

#### 1. nodes/chinese-chess.js
- 前端入口判断逻辑：支持"前"、"后"开头的走法
- 原逻辑：只检查第一个字符是否为棋子名
- 新逻辑：同时检查标准格式（棋子名开头）和方位词格式（前/后+棋子名开头）

#### 2. resources/chinese-chess-core.js
**Game.prototype.move() 函数：**
- 走子前获取棋子信息
- 遍历棋盘找所有同类型同色棋子
- 按列分组，判断起点列是否有多个棋子
- 根据 y 坐标判断"前"还是"后"
- 保存 `positionPrefix` 到 moveObj

**coordinateToChineseMove() 函数：**
- 简化逻辑，直接使用 `move.positionPrefix`
- 不再需要传 game 参数
- 不再需要"把棋子加回去"的复杂逻辑
- 只要有前缀就保留列号（5字符格式）

### 测试用例
| 场景 | 预期输出 | 实际输出 |
|------|---------|---------|
| 两炮在不同列 | 炮二平五 | 炮二平五 ✓ |
| 两炮在同列，走上面的炮（红方） | 前炮二平五 | 前炮二平五 ✓ |
| 两炮在同列，走下面的炮（红方） | 后炮二平五 | 后炮二平五 ✓ |
| 黑方双车同列，走下面的车（靠近红方） | 前车1平2 | 前车1平2 ✓ |
| 黑方双车同列，走上面的车（黑方底线） | 后车1平2 | 后车1平2 ✓ |

### 关键技术点
1. **时机选择**：判断必须在走子前完成
2. **方位判断**：红方y小=前，黑方y大=前（更靠近对方）
3. **格式统一**：只要有前缀就保留列号（5字符格式）
4. **向后兼容**：旧版 history 没有 positionPrefix 也能正常工作

## 下一步开发计划

### 待开发功能：WebSocket 接口
**目标**：通过 WebSocket 接收外部走子命令和 FEN 局面

#### 功能需求
1. **WebSocket Server**：在节点库内启动 WebSocket 服务
2. **接收 Move Payload**：接收走子命令（fromX, fromY, toX, toY 或中文记谱）
3. **接收 FEN Payload**：接收 FEN 字符串并加载到棋盘
4. **状态同步**：走子后向前端同步棋盘状态

#### 接口设计草案
```javascript
// Move 格式
{
  type: 'move',
  from: [7, 7],    // 起点坐标
  to: [4, 7],      // 终点坐标
  notation: '炮二平五'  // 或使用中文记谱
}

// FEN 格式
{
  type: 'fen',
  fen: '3k5/4a4/4c4/9/9/9/9/4C4/4A4/3K5 w - - 0 1'
}

// 响应格式
{
  success: true/false,
  message: '成功/错误信息',
  fen: '当前局面FEN',
  pgn: '当前PGN',
  currentTurn: 'red/black'
}
```

#### 开发步骤
1. [ ] 安装 ws 库依赖
2. [ ] 在 chinese-chess.js 中创建 WebSocket Server
3. [ ] 实现消息解析和分发
4. [ ] 实现 move 类型处理（调用 game.move）
5. [ ] 实现 fen 类型处理（调用 game.loadFEN）
6. [ ] 添加错误处理和响应机制
7. [ ] 前端状态同步（可选）
